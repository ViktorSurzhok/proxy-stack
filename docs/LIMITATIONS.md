# Known limitations — proxy-stack

## Платформа

- Официально ориентир: **Ubuntu / Debian** с `systemd`, `apt`, поддержкой UFW.
- На минимальных образах Ubuntu пакет **`docker-compose-plugin`** иногда отсутствует в apt. Установщик тогда подтягивает **Docker Compose v2** с [релизов docker/compose](https://github.com/docker/compose/releases) в `/usr/local/lib/docker/cli-plugins/` (версию можно переопределить переменной `DOCKER_COMPOSE_PLUGIN_VERSION`).
- Образ **MTProto** с Docker Hub ориентирован преимущественно на **amd64**. На **ARM** (Graviton, Raspberry Pi VPS) `docker pull` может не найти подходящий манифест — потребуется другой образ или сборка из исходников.

## Сеть и IP

- Определение «публичного» IP через `api.ipify.org` / `ifconfig.me` может **не сработать** при офлайн-установке или жёстких egress-фильтрах — тогда используется первая адресная строка `hostname -I` (может быть приватной за NAT).
- Автоподстановка IP для Tinyproxy из `SSH_CONNECTION` — это IP **на момент установки**. Смена домашнего IP потребует `reconfigure.sh` с новым `ALLOWED_IPS_OVERRIDE`.

## Tinyproxy

- Это **HTTP-прокси**: для трафика браузера подходит типичный «HTTP proxy»; не путать с SOCKS5.
- **HTTPS** в браузере обычно идёт через `CONNECT`; Tinyproxy это поддерживает, но политики корпоративных сетей/DPI могут мешать независимо от прокси.
- Полная замена **аутентификации** на BasicAuth без привязки к IP даёт поверхность для перебора паролей — в этом проекте такой режим намеренно не поощряется.

## MTProto

- Официальный контейнер рекомендует периодический **рестарт** (изменения списков core-серверов Telegram). Можно добавить в cron `docker restart proxy-stack-mtproto` (вне scope минимальной установки).
- Занятость порта **443** обрабатывается fallback на **8443**; клиентам нужна ссылка/summary с актуальным портом.

## UFW

- Если файрвол уже был настроен иным инструментом (`iptables-nft` вручную, другие скрипты), возможны **дублирующиеся или конфликтующие** правила вне зоны ответственности `proxy-stack`.

## Идемпотентность

- Повторный `install.sh` **перезаписывает** `/etc/tinyproxy/tinyproxy.conf` своей версией (резервная копия до первой установки: `tinyproxy.conf.bak.before-proxy-stack`).
- Секрет MTProto **сохраняется** между переустановками, пока существует `secrets.env`.
